# Быстрый запуск для локального тестирования

## Сборка проекта

```bash
mvn clean package
```

## Запуск (локально на одной машине)

### Терминал 1 - Сервер

```bash
java -cp target/jade-delivery-system.jar server.ServerLauncher 1099
```

**Что происходит:**
- Создается главный контейнер JADE
- Запускаются агенты на сервере:
  - `coordinator` - координатор системы
  - `warehouse` - склад
  - `store_STORE_001`, `store_STORE_002` - первые 2 магазина
  - `TRUCK_001_server`, `TRUCK_002_server`, `TRUCK_003_server`, `TRUCK_004_server` - первая половина грузовиков

### Терминал 2 - Клиент (через 2-3 секунды после сервера)

```bash
java -cp target/jade-delivery-system.jar client.ClientLauncher
```

Или с явными параметрами:
```bash
java -cp target/jade-delivery-system.jar client.ClientLauncher localhost 1099 Client1
```

**Что происходит:**
- Создается клиентский контейнер, подключается к серверу
- Запускаются агенты на клиенте:
  - `store_STORE_003` до `store_STORE_012` - остальные магазины (10 магазинов)
  - `TRUCK_005_client`, `TRUCK_006_client`, `TRUCK_007_client`, `TRUCK_008_client` - вторая половина грузовиков

## Взаимодействие агентов

### Последовательность работы:

1. **Магазины отправляют запросы координатору:**
   ```
   [store_STORE_001] → Отправлен запрос на доставку координатору
   [coordinator] ← Получен запрос от store_STORE_001
   ```

2. **Координатор планирует маршруты:**
   ```
   [coordinator] Начинается планирование маршрутов...
   [coordinator] Маршруты спланированы: 8
   ```

3. **Координатор отправляет маршруты грузовикам:**
   ```
   [coordinator] → Отправлен маршрут ROUTE_1 грузовику TRUCK_001_server (3 остановок)
   [TRUCK_001_server] ← Получен маршрут от координатора
   [TRUCK_001_server] → Отправлено подтверждение координатору
   ```

4. **Грузовики выполняют доставку:**
   ```
   [TRUCK_001_server] === Начинаю выполнение маршрута: ROUTE_1 ===
   [TRUCK_001_server] → В пути к первой остановке...
   [TRUCK_001_server] ✓ Маршрут выполнен успешно!
   [TRUCK_001_server] → Отправлен отчет координатору
   ```

5. **Координатор получает отчеты:**
   ```
   [coordinator] ✓✓✓ Получен отчет о завершении маршрута
   ```

## Результаты

После выполнения в папке `output/` будут созданы:
- `schedule.csv` - расписание доставки в текстовом формате
- `schedule.xlsx` - расписание доставки в Excel формате

## Данные

- **12 магазинов** (STORE_001 до STORE_012)
- **8 грузовиков** (TRUCK_001 до TRUCK_008)
- **15 товаров** (PROD_001 до PROD_015)
- **Разделение:**
  - Сервер: 2 магазина + 4 грузовика
  - Клиент: 10 магазинов + 4 грузовика

## Логирование

Все взаимодействия между агентами видны в консоли:
- `→` - отправка сообщения
- `←` - получение сообщения
- `✓` - успешное выполнение
- `✗` - ошибка

## Остановка

Нажмите `Ctrl+C` в обоих терминалах для остановки.
